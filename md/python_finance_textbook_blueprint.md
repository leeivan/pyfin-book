# 《Python 程序设计（以金融为背景）》教材蓝图

## 1. 文档定位

本蓝图用于“章节语义建立阶段”，目标是先固定知识递进与教学结构，再进入分章正文写作。  
当前文档不包含章节正文，不替代教材内容稿。

## 2. 读者画像与课程定位（初始假设）

- 目标读者：财经类、管理类、金融工程类本科生（Python 基础薄弱或零基础）。
- 金融背景：具备基础金融常识（收益、风险、贴现、资产类别），但不要求量化交易经验。
- 编程基础：默认从基础语法讲起，逐章叠加到数据分析、策略与风险模拟。
- 课程形态：建议 48-64 学时（理论+实验），支持“课堂讲授 + 机房实操 + 课程项目”。
- 培养目标：让学生具备“可落地的金融编程能力”，而非仅停留在语法记忆层面。

## 3. 全书能力目标

完成本书后，读者应能：

1. 使用 Python 完成常见金融计算（利率、终值、现值、收益率、风险指标）。
2. 使用结构化编程与函数封装构建可复用的金融小工具。
3. 使用文件与表格数据完成金融数据读取、清洗与基础分析。
4. 使用 NumPy/Pandas/Matplotlib 完成时间序列分析与可视化表达。
5. 完成从“策略原型 -> 回测评估 -> 风险分析 -> 系统化实现”的入门闭环。

## 4. 知识依赖与递进路径

全书依赖主链：

`第1章认知与环境 -> 第2章语法基础 -> 第3章控制逻辑 -> 第4章数据组织 -> 第5章函数模块化 -> 第6章数据读写 -> 第7章面向对象 -> 第8章分析库 -> 第9章策略 -> 第10章风险模拟 -> 第11章系统接口 -> 第12章综合项目`

关键跨章依赖：

- 第5章是第9章与第10章的“函数复用前置”。
- 第6章与第8章共同构成第9章策略实验的数据基础。
- 第7章为第11章系统设计提供对象抽象方法。
- 第9章（策略）与第10章（风险）共同汇入第12章综合项目。

## 5. 分章语义规划（12章）

### 第1章 Python 与金融计算概述
- 本章定位：建立学习动机，回答“为什么金融专业要学 Python”。
- 前置依赖：无。
- 关键能力：完成开发环境搭建与首个金融计算程序。
- 章末交付物：`simple_return_calculator.py`（简单收益计算脚本）。

### 第2章 Python 基础语法与金融实例
- 本章定位：建立变量、表达式、输入输出的操作闭环。
- 前置依赖：第1章开发环境。
- 关键能力：将金融公式正确映射为代码表达式。
- 章末交付物：迷你“金融报表打印程序”。

### 第3章 程序控制结构与金融逻辑
- 本章定位：把静态计算升级为“可决策、可迭代”的程序。
- 前置依赖：第2章语法基础。
- 关键能力：用条件与循环实现贷款还款计划。
- 章末交付物：贷款还款计划表生成器（控制流版本）。

### 第4章 复合数据类型与金融数据组织
- 本章定位：建立金融数据在内存中的组织模型。
- 前置依赖：第3章控制结构。
- 关键能力：使用列表/字典/集合管理多资产数据。
- 章末交付物：多资产收益数据结构化管理脚本。

### 第5章 函数与模块化金融程序设计
- 本章定位：从“能跑”过渡到“可复用、可维护”。
- 前置依赖：第4章数据结构。
- 关键能力：封装金融计算函数并组织为模块。
- 章末交付物：`finance_utils/` 计算工具库（利率、现值、终值函数）。

### 第6章 文件操作与金融数据读写
- 本章定位：打通程序与真实金融数据文件之间的输入输出通道。
- 前置依赖：第5章函数模块化。
- 关键能力：读取 CSV、完成基础清洗并保存结果。
- 章末交付物：历史交易数据导入与预处理脚本。

### 第7章 面向对象程序设计与金融建模
- 本章定位：用类抽象金融对象，建立建模思维。
- 前置依赖：第5章函数抽象、第4章数据组织。
- 关键能力：实现股票/债券/基金类及投资组合对象。
- 章末交付物：投资组合对象模型（含基础方法）。

### 第8章 常用金融数据分析库
- 本章定位：引入数据分析工业工具链。
- 前置依赖：第6章文件数据、第2章语法。
- 关键能力：用 NumPy/Pandas/Matplotlib 完成价格序列分析。
- 章末交付物：股票价格走势分析 Notebook。

### 第9章 金融算法与策略初步
- 本章定位：从数据分析进入策略原型设计。
- 前置依赖：第5章函数、第8章数据分析库。
- 关键能力：实现移动平均策略并执行基础回测。
- 章末交付物：简单量化交易策略脚本（含绩效指标输出）。

### 第10章 金融风险分析与模拟
- 本章定位：将“收益思维”扩展为“收益-风险联合思维”。
- 前置依赖：第8章数据分析、第9章策略评估。
- 关键能力：波动率度量、Monte Carlo 模拟、情景分析。
- 章末交付物：资产收益分布模拟与风险报告。

### 第11章 金融系统与接口应用
- 本章定位：面向实务系统化，连接外部数据与内部逻辑。
- 前置依赖：第6章数据读写、第7章对象建模。
- 关键能力：API 调用、异常处理、查询系统实现。
- 章末交付物：简版金融数据查询系统（可查询+可容错）。

### 第12章 课程综合项目
- 本章定位：完成课程级综合实践，验证知识迁移能力。
- 前置依赖：前11章。
- 关键能力：需求分析、架构设计、实现测试与项目展示。
- 章末交付物：综合项目（建议：投资分析系统）。

## 6. 全书统一章节写作协议（后续正文强制执行）

每章正文必须按以下 6 个区块顺序展开：

1. 导入：现实问题（约 10%）
2. 直觉理解（约 20%）
3. 概念与规则（约 25%）
4. 机制或原理（约 15%）
5. 应用与案例（约 20%）
6. 规范与风险（约 10%）

每个知识点必须覆盖 8 个教学组件：

1. 动机
2. 直觉
3. 错误直觉
4. 定义
5. 示例
6. 错误示例
7. 实际意义
8. 小结

## 7. 分批写作与规模控制（后续执行规则）

- 单次产出单位：仅一个区块，约 2000-3000 字。
- 章节目标规模：建议 20000-50000 字（按课程需求可裁剪）。
- 深化触发：当区块解释不足、术语跳跃或示例过少时，进入教学深化模式。
- 禁止行为：摘要式压缩、无信息重复、跳过认知建立过程。

## 8. 附录规划

附录 A：常用金融术语与公式  
附录 B：Python 常用函数速查表  
附录 C：实验与项目参考数据说明
