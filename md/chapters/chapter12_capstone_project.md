# 第12章 课程综合项目（章节文档）

## 1. 章节定位

本章通过综合项目检验全书能力迁移，形成“问题定义 -> 系统实现 -> 结果评估 -> 展示复盘”的完整工程闭环。

## 2. 目录结构

1. 项目需求分析与设计
2. 金融场景综合实践（如：投资分析系统）
3. 项目实现与测试
4. 项目展示与课程总结
5. 本章小结与能力复盘

## 3. 内容介绍

### 3.1 项目需求分析与设计
- 确定项目目标、目标用户和功能范围。
- 输出需求文档、模块划分与里程碑计划。
- 明确数据来源、性能要求和风险约束。

### 3.2 金融场景综合实践
- 推荐主题：投资分析系统、组合监控系统、策略评估系统。
- 要求整合数据读取、指标计算、策略分析与风险评估。
- 强调问题驱动，而不是功能堆叠。

### 3.3 项目实现与测试
- 按模块分工并进行版本管理。
- 完成功能测试、边界测试和异常测试。
- 输出测试记录和缺陷修复说明。

### 3.4 项目展示与课程总结
- 进行项目演示：问题背景、方法、结果、局限与改进方向。
- 解释关键技术决策和金融业务价值。
- 完成团队复盘与个人能力总结。

## 4. 实验与实践建议

- 实验 1：提交项目立项书和原型设计图。
- 实验 2：完成中期演示并接受同伴评审。
- 实验 3：完成终期答辩与代码文档归档。

## 7. 教学提示

- 本章核心是综合能力，不建议只以代码行数评分。
- 建议评分维度覆盖：需求合理性、技术实现、金融解释、测试完备度、展示质量。
