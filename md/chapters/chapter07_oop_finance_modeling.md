# 第7章 面向对象程序设计与金融建模（章节文档）

## 1. 章节定位

本章引入类与对象，把离散函数升级为可扩展的金融对象模型。

## 2. 目录结构

1. 类与对象基础
2. 金融产品类设计（股票、债券、基金）
3. 封装、继承与多态在金融系统中的应用
4. 案例：投资组合对象建模
5. 本章小结与模型扩展讨论

## 3. 内容介绍

### 3.1 类与对象基础
- 介绍类属性、实例属性与方法。
- 通过“一个资产对象”示例解释状态与行为。
- 强调构造函数参数与对象有效性校验。

### 3.2 金融产品类设计
- 定义 `Stock`、`Bond`、`Fund` 等基础类。
- 抽象共同字段（代码、名称、持仓量、估值方法）。
- 讨论不同产品的收益与风险特征差异。

### 3.3 封装、继承与多态
- 用继承减少重复代码，统一产品接口。
- 用多态实现“同一调用，不同产品行为”的估值计算。
- 强调过度继承风险与组合优先原则。

### 3.4 案例：投资组合对象建模
- 构建 `Portfolio` 类，管理多资产持仓。
- 实现组合市值、组合收益率等方法。
- 为后续策略和风险章节预留接口。

## 4. 实验与实践建议

- 实验 1：实现三个金融产品类并实例化测试。
- 实验 2：实现组合类并支持资产增删改查。
- 实验 3：扩展一个新产品类（如 ETF）验证模型可扩展性。

## 5. 本章交付物

- `financial_products.py`
- `portfolio_model.py`

## 6. 与前后章节衔接

- 前置依赖：第5章模块化思想与第4章数据结构。
- 后续衔接：第11章系统设计将复用对象模型组织业务逻辑。

## 7. 教学提示

- 本章应避免“只讲语法继承”，要突出业务抽象价值。
- 建议评估维度包含：可扩展性、接口一致性、业务语义清晰度。
